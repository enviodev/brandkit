6. Content Guidelines

Documentation and Technical Writing

- Use second-person ("you") to speak directly to the developer.
- Structure content with clear headings, step-by-step instructions, and working code examples.
- Every code example should be copy-paste ready and tested.
- Link generously to related docs, GitHub issues, and community channels.
- Use callouts to highlight warnings, tips, and important notes.

Social Media

- Twitter / X: Developer-first tone. Share updates, ecosystem wins, technical tips. Short, punchy. Emoji acceptable but not required.
- Discord: Community-first. Warm, helpful, fast to respond. Celebrate contributors and milestones.
- LinkedIn: Slightly more formal but still approachable. Share product updates, team news, ecosystem partnerships.
- GitHub: Code speaks loudest. READMEs and changelogs should be clear, thorough, and current.

Announcements and Press

- Lead with the developer or user benefit, not the internal feature name.
- Anchor claims in data where possible, for example: "2x faster sync", "supports 30+ EVM chains".
- Avoid embargoed information in community channels.
